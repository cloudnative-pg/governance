## The "Human-First" Contribution Disclaimer

CloudNativePG is a community-driven project. While many maintainers are
sponsored by their respective employers, they operate here on a voluntary
basis.

> [^IMPORTANT]
> **We are all human beings with finite time and a limited amount of cognitive
> load we can sustain at any given moment.** Given the rapid growth and technical
> complexity of the project, maintainers must carefully balance planned roadmap
> items and critical maintenance (such as the current transition of Barman Cloud
> to plugins and moving from base-image extensions to self-contained ones).

To respect everyone's time, please follow these steps before submitting a Pull
Request:

1.  **Open an Issue**: Always start by opening a GitHub Issue to discuss your
    idea.
2.  **Wait for Alignment**: Ensure maintainers have agreed on the approach and
    the priority before you start coding.
3.  **Experimental Path**: If your patch cannot wait or is highly experimental,
    remember that the **CNPG-I interface** provides the freedom to build and test
    features as plugins. Once a feature is proven and well-received by the
    community, we can discuss moving it into the core.

We welcome many types of contributions including:

    - [Sources](https://github.com/cloudnative-pg/cloudnative-pg/tree/main/docs)

- [Communications / Social Media](https://github.com/cloudnative-pg#getting-in-touch) / [Blog Posts](https://cloudnative-pg.io/blog/)

## General Workflow

1.  **Ask for Help**: The best way to reach us is via our [Slack channel or by starting a GitHub discussion](https://github.com/cloudnative-pg#getting-in-touch).

2.  **Raising Issues**: When raising issues, please use the provided templates
    and include:
    - Setup details and environment information.
    - A reproduction scenario.
    - Relevant logs and error messages.
3.  **Security**: If you are reporting a vulnerability, please refer to the
    [Security Policy](https://github.com/cloudnative-pg/cloudnative-pg/blob/main/SECURITY.md).

## External Contributors vs Maintainers

**External Contributors:** If you're contributing to the CloudNativePG operator
from outside the core team, please note that some instructions in our detailed
development docs apply only to maintainers.
See the [development contribution guide](https://github.com/cloudnative-pg/cloudnative-pg/blob/main/contribute/README.md)
for complete details, but note:

- **Issue Assignment**: Comment "I'd like to work on this" on an issue rather than self-assigning.
- **Testing**: Run local tests before submitting. Repository-specific testing
  instructions can be found in the respective `CONTRIBUTING.md` or
  `contribute/` folders of each repo.
- **Project Boards**: Maintainers handle the movement of tickets through
  project phases.
